# BTCDayu 参考站改写闭环文档

目标
- 不是继续讨论“这个站有没有帮助”。
- 是直接把参考结论落成可执行、可发布、可扩写的站点版本。
- 输出对象：当前仓库里的 USDT 新手手册。

结论
- BTCDayu 的核心价值不在单篇文章质量，而在“把新手一步一步带进去”的路径设计。
- 这套路径值得借。
- 返佣味、标题躁感、分类混排，不值得借。

一、参考站真正值得保留的骨架

参考站的强项，可以抽象成 4 层：
1. 导航与入门
2. 作者信任层
3. 长尾知识沉淀层
4. 工具 / 项目 / 场景延伸层

但当前项目不需要照搬整站风格。
我们只保留最有效的部分：
- 用“步骤感”而不是“资料堆砌”做首页
- 用“新手任务”而不是“行业大类”做主导航
- 用“场景页”承接搜索词，再自然推荐钱包工具

二、参考站到当前项目的映射

BTCDayu 原栏目 -> 当前项目的改写方式

1. 币圈新手入门
- 保留
- 改写为更克制、更可信的手册路径：
  - Start Here

  - 怎么选钱包

- 不保留原表达
- 替换为：
  - beginner-friendly tools
  - use cases

- 原因：当前站点目标是教程信任，不是高刺激转化。

3. 作者原创 / 长文 / 书单 / 投资经典
- 当前阶段不作为主导航
- 放入未来扩展层：
  - 认知与原则
  - 钱包评测方法
  - 编辑原则
- 原因：首页先解“怎么做”，不是先讲“我懂很多”。

4. 播客 / 项目研究
- 当前阶段延后
- 优先做能承接搜索意图的实用专题页

四、当前项目对参考站做过的关键修正

1. 首页从“转化页”改成“手册页”
- 重点不再是按钮和推荐码
- 重点是阅读顺序、章节结构、风险说明

2. 工具推荐从“主角”降为“上下文里的建议”
- imToken 出现方式：
  - beginner-friendly option
  - 适合谁

- 不作为首页唯一主 CTA

3. 增加 trust 层
- `how-we-review-wallets-and-apps.html`
- `editorial-policy.html`
- 这部分是参考站相对薄弱、但当前项目必须补齐的地方

4. 把“币圈感”降到“用户任务感”
- 用“怎么买、怎么发、怎么选、怎么付”替代“神级工具、暴赚、教程神话”
- 好处：
  - 更适合搜索
  - 更适合第三方教程站定位
  - 更利于长期留存与扩展

七、标题风格规则

保留参考站的“明确解决问题”能力。
去掉参考站的“刺激性、收益感、营销味”。

推荐标题风格
- 怎么买 USDT
- 怎么选钱包
- 为什么转 TRC20 USDT 要准备 TRX
- 如何用 USDT 订阅 AI
- 出国旅游时，USDT 有哪些实际用途

避免标题风格
- 神级工具
- 暴赚入门
- 从零到几百万
- 40% 年化
- 一次全搞定

八、写作规则

每篇文章默认包含：
1. 这篇解决什么问题
2. 先解释概念
3. 给出默认路径
4. 列出最容易踩的坑
5. 给 1 个 beginner-friendly option
6. 放下一篇阅读建议
7. 加风险提醒

这样写的原因很简单：
- 新手不是缺观点，是缺顺序
- 新手不是缺术语，是缺“下一步点哪里”
- 新手不是缺工具，是缺不翻车的默认路径
