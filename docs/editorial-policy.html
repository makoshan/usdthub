编辑原则

  1. 先解释问题，再推荐工具。如果一段文字里出现了钱包、交易所或服务推荐，前面一定先说清楚「你要解决的是什么问题」「它适合谁」「它不适合谁」。
  2. 所有风险场景统一加风险提醒。买币、转账、租能量、签名授权、C2C、跨境支付这几类场景里，页面都会有 .risk 样式的风险块。
  3. 推荐必须三件事齐全：适合谁、为什么适合、局限是什么。只写"推荐"两个字的段落视为不合格，要重写。
  4. 禁用夸张语："绝对安全"、"全网最好"、"适合所有人"、"零风险"、"闭眼买"——这些词在加密领域基本等于警报，站内一经发现必须改写。
  5. YMYL 原则：涉及资金、私钥、合规、账户风控的内容，不采用任何"博眼球"的写作技巧。

内容生产与审校流程

  1. 起草：我用 Markdown 在本地写草稿（仓库里的 markdown/ 目录就是源文件）。
  2. 自审：按下列清单检查——a) 有没有在推荐前解释问题；b) 有没有给出实际限制和局限；c) 所有链接是否能打开、是否是官方或一手来源；d) 风险提醒是否写到位；e) 有没有任何夸张语。
  3. commit + push：发布到 GitHub，触发 Jekyll 构建，部署到 docs/。
  4. 读者纠错：收到事实性错误反馈后，优先级按"影响读者资金安全 > 影响操作正确性 > 文字打磨"排序；前两类承诺 48 小时内处理。
  5. 定期复审：涉及具体数字（TRON 能量费、交易所手续费、KYC 要求）的页面每季度至少复审一次；发现数据过期立即更新。
