How We Review Wallets and Apps

Trust page · 评测方法与评分体系

这个手册是第三方教程站，不是交易平台，也不是"谁给得多就写谁"的榜单页。我们评测钱包和应用时，优先看的是：新手能不能理解、能不能安全完成任务、以及它是否真的适合 USDT 的实际使用路径。

短版结论
我们不按 payout 大小排推荐顺序。我们按是否更容易理解、是否更适合新手、是否更适合当前任务来排序。目前站内没有任何返佣链接、没有邀请码、没有赞助内容——详情见 商业披露表。

评分体系：5 个维度 + 权重

每个钱包 / App 按 5 个维度打分（0–10 分），加权汇总成"新手适配度"。维度和权重如下：

维度权重这一项看的是
新手友好度 25% 第一次安装到完成"收 / 发一笔 USDT"全流程是否清楚。图标有没有歧义、文案有没有绕、错误提示有没有写清楚发生了什么。
自托管清晰度 25% 是否真的把控制权交给用户、助记词生成流程是否透明、备份提示是否足够刺眼、恢复流程是否真的可走通（我们会拿副本设备验证恢复一次）。
TRON / USDT 路径 20% TRC20 USDT 收发是否有"选错网络"保护、能量 / 带宽不足时是否有提示、USDT 的 Memo / Tag 机制是否解释清楚。
风险提示充分度 15% 签名授权时是否解释了你在签什么、大额转账是否有二次确认、钓鱼 dApp / Drainer 场景下是否有警报。
产品成熟度 15% 活跃开发（最近 3 个月是否有更新）、公开口碑（社区投诉集中在什么问题）、事故响应（过去是否发生过安全事件、处理方式是否公开）。

单一维度得分低于 4 分会被标记为"新手不推荐"，即使其他维度都很高。这个设计是为了把新手最容易掉进去的坑优先排除。

测试流程

每个上榜的钱包 / App 都经过以下流程，由作者本人实际操作：

  1. 干净设备 + 新身份：在一台全新或重置的手机 / 浏览器 profile 上完成整个测试，模拟"第一次用"的真实场景。
  2. 实际操作四件事：a) 生成钱包 + 抄写助记词；b) 收一笔小额 TRC20 USDT；c) 发一笔小额 TRC20 USDT（含能量不足场景）；d) 在副本设备上用助记词恢复验证。
  3. 记录每个卡点：哪里看不懂、哪里让我不安全地 approve、哪里没有明确提示。这些成为评分的依据。
  4. 看一圈公开投诉：GitHub Issue、Reddit、Twitter / X、以及中文社区（知乎、V2EX、Telegram）过去 12 个月的高热度投诉集中在什么问题。
  5. 第二轮确认：间隔 2–4 周后，再跑一次关键路径，确认不是首次印象偏差。

更新节奏

  • 每季度至少复审一次站内已上榜的钱包 / App，确认版本号、关键功能、风控提示没有大幅回退。
  • 遇到以下情况立即重写评测：a) 发生公开的安全事件；b) 产品大改版影响了新手流程；c) 公司 / 开发方发生重大变动（被收购、核心团队离场等）。
  • 每次版本更新会在 Changelog 里记录，并更新文章底部的dateModified。

推荐会以什么方式出现

  • editor's pick——按上面 5 个维度综合最高分；
  • beginner-friendly option——新手友好度 + 自托管清晰度特别突出；
  • easiest path for a specific task——针对某个具体任务（例如"只需要收一笔 USDT 然后转到交易所"）最顺手的一款。

也就是说，推荐会放在具体上下文里，不会上来就把某个钱包写成"全站唯一主角"。

利益冲突处理

  • 作者目前未从任何钱包厂商 / 交易所收取返佣、评测费、赠品、股权或顾问收入——详见 商业披露表。
  • 如果作者本人长期使用某款钱包（例如 imToken / Ledger 作为日常工具），文章内部会明确披露"我自己用 X 作为主钱包"，并不因为日常使用就自动把它推成"最好"。
  • 钱包厂商 / 交易所不能通过付费来加快评测、影响评分、或删除不利描述。收到此类请求会直接拒绝并在 GitHub Issue 里公开记录（如请求方同意公开）。

我们不做什么

  • 不作为交易平台；
  • 不提供投资建议；
  • 不代用户买 USDT 或代管资产；
  • 不承诺某一个钱包适合所有人；
  • 不对读者自己操作中的签名授权 / 钓鱼 / 社工损失负责——评测不能替代读者本人的风险判断。
重要提醒
硬件钱包 / 软件钱包的"评分高"不等于"你用它就一定安全"。公开的链上案例里，绝大多数钱包资产丢失是因为助记词泄露（截屏上传 / 云盘备份 / 钓鱼网站骗输入）和签名授权被骗（假 dApp / Drainer 合约）。无论用哪一款钱包，都请遵守：助记词线下备份、不在联网设备截屏 / 上传、对所有"代签 / 代恢复 / 领空投" 的私信保持警惕。
