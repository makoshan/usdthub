# 内容风格说明

目标
- 参考 BTCDayu 的内容格式和口味，但不照搬它的返佣感和标题躁感。
- 保留它最值钱的部分：第一人称、路径感、实操感、真干货。

一句话原则
- 像一个真的做过这些事的人在带路。
- 不是百科，不是公关稿，不是模板文。

一、统一口吻

必须做到
- 第一人称：多用“我”“我建议”“我自己的做法”“我更看重”。
- 直接判断：不要假装中立到没有立场。
- 讲人话：少术语堆砌，先把事讲明白。
- 讲代价：说方法的时候，同时说坑和边界。

避免
- “本平台认为”“用户可根据自身情况选择”这种空话。
- “最强”“闭眼冲”“稳赚”这种垃圾转化话术。
- 纯客观说明书口吻，像机器在复读定义。

二、文章结构

每篇默认按这个顺序写：
1. 先说这篇解决什么问题
2. 我为什么这样看
3. 默认路径怎么走
4. 哪些坑最常见
5. 我给新手的建议
6. 风险提醒
7. 下一篇读什么

三、格式要求

推荐格式
- 开头先下判断
- 中间用小标题拆解
- 每节尽量有一句“我自己的结论”
- 能列步骤就列步骤
- 能列坑就列坑

推荐句式
- 我先说结论：
- 我自己的经验是：
- 如果你是新手，我建议你先这样做：
- 这件事真正容易翻车的地方在于：
- 我不建议你一上来就：

四、干货标准

什么叫真正的干货
- 不是定义更多
- 是读完知道下一步该做什么
- 是知道哪一步最危险
- 是知道为什么很多人会在这里翻车
- 是知道什么情况该停手

不算干货的东西
- 只解释概念，不给路径
- 只给路径，不说风险
- 只讲优点，不讲限制
- 只有“推荐工具”，没有“为什么推荐”

五、与 BTCDayu 相同和不同的地方

保留
- 第一人称
- 带路感
- 直接下判断
- 明显的“这是做过的人在写”

不保留
- 强返佣味
- 过躁标题
- 太重的“暴赚 / 神级工具 / 一次全搞定”口吻

六、推荐口径

可以写
- 这是我愿意给新手的一个起步选项
- 如果你只是想先把流程跑通，我更建议你用这个
- 它不一定适合所有人，但对新手更省脑子

不要写
- 全网最佳
- 适合所有人
- 不用想直接上

七、首页和核心文章的额外要求

首页
- 要像手册首页，不像广告页
- 要让我这个作者的判断露出来
- 要告诉读者：我会带你从哪里开始

核心文章
- `what-is-usdt.html`
- `how-to-buy-usdt.html`
- `how-to-send-usdt.html`
- `usdt-networks-explained.html`
- `tron-energy-guide.html`

这些页面必须最像“真人在带路”，不能写成百科词条。
