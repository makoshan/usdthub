# 关于作者 / About

> TL;DR · 我叫 Mako Shan。这个站是一份公开笔记，不是交易平台，也不是投资建议。我写它是因为我自己第一次用 USDT 的时候没有一份像样的、用中文说清"到底怎么做才不容易出事"的手册。它不是匿名运营的网站，所有内容都在 GitHub 上公开版本控制。

## 我是谁

我是 Mako Shan，一个在独立做开发和内容的人。我的日常背景在软件工程和产品，不代表任何交易所或钱包厂商。跟加密的关系来自两件事：

1. 我自己用 USDT 做跨境支付、订阅海外服务（ChatGPT、eSIM、VPN、云主机）以及少量长期持仓；
2. 身边长期有**真的第一次接触 USDT / 硬件钱包 / P2P** 的朋友，他们踩过的坑和问我的问题，慢慢变成了这份手册的章节。

我在 GitHub 上以 [@makoshan](https://github.com/makoshan) 发布代码和工具，这个站的源代码也在同一个账号下公开（[makoshan/usdthub](https://github.com/makoshan/usdthub)），每一次修改都有 commit 历史可查。

## 我为什么写这个站

第一次用 USDT 的人，真正缺的是**一段把路讲明白的中文**。2024–2026 这几年，中文世界里的 USDT 资料普遍有三个问题：

- **推广味太重**——所有页面都在把你往某个注册返佣链接推，没有在解释你在做什么；
- **对中国大陆用户的现实问题避而不谈**——支付风控、冻卡、问题资金、解释成本，这些才是真正会伤到新手的东西；
- **把复杂的东西讲得更复杂**——硬件钱包、TRON 能量、TRC20 vs ERC20，这些概念其实都可以讲清楚，不需要夹杂行话。

这个站存在的唯一目的，是把**第一次操作**这段路写清楚。不解决投资决策，不判断行情，也不告诉你该不该买币。

## 我怎么写

我写内容的原则和质量流程记录在 [Editorial Policy](./editorial-policy.md) 和 [How We Review Wallets and Apps](./how-we-review-wallets-and-apps.md) 两页。核心几条：

- **先解释问题，再推荐工具**——如果你看到我推荐某个钱包或平台，前面一定先讲了它适合谁、为什么适合、局限是什么。
- **冷钱包、交易所、P2P 商家我都实际用过**——都是我自己的真实使用场景。具体到某些细节我会注明数据来源（比如官方文档 / 我实测 / 朋友反馈）。
- **每一次更新都有 git 记录**——页面上显示的"最近更新"由构建脚本直接从 git log 读出来，不会出现"全站统一写成今天"这种粉饰更新频率的情况。
- **YMYL 原则**——加密 / 金融属于 Google "Your Money or Your Life" 范畴，这类内容容错空间比普通话题小得多。凡涉及风险（冻卡、问题资金、私钥丢失、钓鱼授权），我都会写明风险边界，不会为了流量去掩饰。

## 商业披露

这个站目前**没有任何返佣链接、没有邀请码、没有付费广告位**。外部链接全部是 `rel="noopener noreferrer"`，不带 utm / ref / 邀请码。如果未来加入任何形式的商业合作，会在 [Editorial Policy](./editorial-policy.md) 里按类别公开披露，并且不会影响*推荐顺序*——推荐顺序始终按"是否适合当前任务 / 是否适合新手"来排。

## 我不做什么

- 我**不提供投资建议**，也不判断币价；
- 我**不代买、不代转、不代管资产**；
- 我**不会主动私信你**——所有自称"Mako"向你要私钥 / 助记词 / 授权的都是假的，任何时候都不要把这些信息给任何人，包括我；
- 我**不承诺某个钱包 / 平台"最好""全网第一""绝对安全"**——这些词在加密领域基本等于警报。
