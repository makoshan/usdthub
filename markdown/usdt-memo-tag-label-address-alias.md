# USDT 的 Memo、Tag、Label、Address Alias 是什么

> Chapter 2.6 · Transfers

新手被交易所和钱包里的英文词搞糊涂，往往不是因为链太复杂，而是因为界面上同时出现了 `memo`、`tag`、`label`、`address alias` 这类词。它们看起来像一回事，实际不完全一样。

> TL;DR：先记最关键的一句。**这些词里，只有一部分是链上真实需要填写的“附加收款信息”，另一部分只是平台自己给地址起的备注名。** 不懂时先停，不要乱填。

## 一、Memo / Tag 通常在说什么


**Memo / Tag** 在交易所语境里通常是同一类东西：它是给平台识别"这笔充值归到哪个账户"的附加信息，常见于共享充值地址或某些特定网络。平台明确要求时，地址和 Memo / Tag 必须一起对。

在很多交易所语境里，`memo` 或 `tag` 的意思很接近：

- 它是给平台识别“这笔钱该记到谁名下”的附加信息
- 常见于共享充值地址或某些特定网络
- 如果平台明确要求你填，漏掉可能会导致到账但不能自动入账

对新手最重要的不是背定义，而是：

**如果充值页面明确给了地址 + Memo / Tag，两者通常都要一起对。**

## 二、Label / Recipient Label 常常只是备注名


**Label / Recipient Label** 在大多数钱包和交易所里，只是给你自己看的备注名——用来给地址起个好记的名字、方便在地址簿或白名单里认出来，通常不是链上字段，而是平台界面的管理字段。

`label` 在很多钱包或交易所里，更多是给你自己看的：

- 给这个地址起个名字，方便下次认出来
- 可能出现在地址簿、白名单地址、常用收款人列表里
- 它常常不是链上字段，而是平台界面的管理字段

所以看到 `recipient label` 时，第一反应不要是“链上多了一个必须填的参数”，而要先判断它是不是平台让你自己写备注。

## 三、Address Alias 又是什么


**Address Alias** 字面上容易被误会成"另一个链上地址"，但大多数时候它只是平台给地址起的别名，比如地址簿昵称、收款模板自定义名称。它到底是不是链上必要字段，要看具体平台和网络设计，不能光看词猜。

`address alias` 更容易让人误解成“另一个链上地址”。很多时候它其实只是：

- 平台给地址起的别名
- 地址簿里的昵称
- 某个收款模板的自定义名称

它是否真是链上的必要字段，完全看平台和网络设计，不能光看词猜。

## 四、TRC20 USDT 一般该怎么理解这个问题


**TRC20 USDT 的日常转账**里，大多数普通用户真正要关心的是网络、地址、金额和 TRX 资源够不够，而不是 memo / tag 本身——这类附加字段在 TRC20 USDT 常规场景里并不常出现。

对大多数普通用户来说，TRC20 USDT 最常见的场景还是：

- 确认网络是 TRON / TRC20
- 复制正确地址
- 确认金额
- 看自己有没有足够 TRX 或资源

也就是说，**你在 TRC20 USDT 的日常转账里，通常更常担心的是地址和网络，而不是 memo/tag 本身。**

## 五、我自己的默认处理规则


**默认处理规则**就一句话：平台没要求的字段不要自作聪明乱填，平台明确要求 memo/tag 的就当整体核对，看到 label/alias 先判断它是不是仅供自己备注，不确定时先去查官方帮助页。

1. 如果平台只给地址，没有要求额外字段，就不要自作聪明乱填
2. 如果平台明确给了地址 + memo/tag，就把它当成一个整体来核对
3. 如果你看到的是 label / alias，先判断它是不是仅供你自己备注
4. 不确定时先查官方帮助页，不要赌
